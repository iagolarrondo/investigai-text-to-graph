Four demo scenarios baked into the data

  • The agent on the claim — a writing agent overlaps with a claimant on a busy policy.
  • Same bank account, different mailboxes — two people share an account from different addresses.
  • The claims sit on a family web — spouse and family ties link people who also appear on claims.
  • Care business at the insured's address — a provider shares an address with multiple members.

3 · How the LLM works today

The user types a question in English. Claude turns it into a structured intent. The graph engine does the rest.

1
User question"Who else is on that claim's network?"
2
System prompt~40 KB of domain schema & examples, cached after the first call.
3
Claude callReturns a JSON intent: {intent, claim_node_id, reason}
4
Graph queryTyped query runs over nodes.csv / edges.csv in NetworkX.
5
Answer + subgraphTables, a plain-English explanation, and a visual subgraph.
Model
claude-opus-4-6   Anthropic API
Pattern
Intent routing → deterministic graph query (no free-form SQL / Cypher generation yet)
Prompt
System prompt with full data dictionary + few-shot examples, marked as an ephemeral cache block (~10% cost after first call)
Output
Strict JSON intent — parsed, validated against a fixed set, fallback to unknown on any error
Grounding
LLM never fabricates IDs — it selects from a known node list or extracts one from the question
Interface
Streamlit app with demo questions, free-text input, and an interactive graph explorer

4 · Next steps

Where we go from a PoC to something an investigator uses every day.

UI & investigator workflow

  • From Streamlit demo to a real investigator console — case-centric, multi-pane.
  • Saveable investigations: pin entities, annotate edges, export a case file.
  • Always-on graph canvas with filters by entity type, time, and risk score.
  • In-line citations — every answer links back to the source rows in the graph.

Evaluation & trust

  • Gold-set evals: ~100 investigator questions with known correct subgraphs.
  • Intent-routing accuracy, retrieval recall, and end-to-end answer correctness.
  • Hallucination guardrails: every entity in an answer must exist in the graph.
  • Human-in-the-loop feedback capture to tune prompts and expand the intent set.

Graph & data depth

  • Scale from 93 nodes to real book volumes — policies, claims, providers, payments.
  • Add temporal edges (when did this link form?) for sequence-based detection.
  • Connect unstructured sources: claim notes, call transcripts, adjuster memos.
  • Entity resolution upgrade — probabilistic matching on people and businesses.

LLM & reasoning

  • Move from fixed intents to tool-use: the model calls typed graph functions.
  • Multi-hop reasoning with agentic loops for "find anyone within 3 hops of…" style asks.
  • Risk scoring on subgraphs — combine graph features with LLM rationale.
  • Pilot on a Manulife business area: disability, LTC, or group benefits fraud.
