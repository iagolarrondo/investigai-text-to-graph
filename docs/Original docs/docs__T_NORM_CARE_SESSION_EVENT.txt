Each entry is a care session ping event that is automatically sent via the Caregiver app. Other tables can be joined to this table via the NORM_CARE_SESSION_ID.

Each event or ping is associated with a timestamp specified by EVENT_LOCAL_TS and an EVENT_TYPE.
Each ping occurs under a care session that is specified by the NORM_CARE_SESSION_ID.
Note that a single care session can have a lot of pings/events or sometimes no events at all based on the ICP's usage of the care giver app the permissions they grant. 

To find the geolocation coordinates for each event/ping join with T_RESOLVED_CARE_SESSION_EVENT_GEOLOCATION_CROSSWALK table on NORM_CARE_SESSION_EVENT_ID.
This result can be further joined with T_RESOLVED_GEOLOCATION table on the RES_GEOLOCATION_ID field to extract the longitude and latitude coordinates.

IMPORTANT: DO NOT search for a claim number using CLAIM_ID field in T_NORM_CARE_SESSION_EVENT table. Join it with T_NORM_CARE_SESSION on NORM_CARE_SESSION_ID, and then JOIN with T_NORM_CLAIM on CLAIM_ID. You can then search for claim number in the CLAIM_NUMBER field of T_NORM_CLAIM.
